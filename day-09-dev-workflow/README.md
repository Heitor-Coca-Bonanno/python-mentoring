# Day 09 — Dev Workflow (Git + clean code)

## Git = version history
Git is a save system for code: each commit is a snapshot you can return to.

## Core workflow
`git status` → `git add` → `git commit` → `git push`

## Good commit messages
- imperative: “Add…”, “Fix…”, “Refactor…”
- specific: say **why** and **what**

## Clean code
Good names, short functions, consistent formatting, and helpful comments.
