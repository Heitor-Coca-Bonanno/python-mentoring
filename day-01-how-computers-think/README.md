# Day 01 — How Computers Think

## Big idea: an algorithm is a recipe
An **algorithm** is a set of step-by-step instructions to solve a problem.

- If your steps are missing something, the result breaks.
- If your steps are ambiguous, different people will do different things.

**Recipe analogy**: “Bake a cake” is not an algorithm. “Preheat oven to 180°C, mix flour + eggs…” is.

## Input → Process → Output (IPO)
Almost every program can be described as:

- **Input**: what the program receives (user typing, button click, file, sensor)
- **Process**: the transformation or logic (calculate, compare, sort, decide)
- **Output**: what the program produces (text, number, saved file, UI change)

**Vending machine analogy**: insert money (input) → machine decides (process) → snack comes out (output)

## Pseudocode: thinking before syntax
**Pseudocode** is writing the algorithm in plain language before writing real code.

Why it helps:
- You can focus on logic without worrying about Python rules yet.
- It’s easier to spot missing steps or unclear parts.
- You can share it with someone and test the algorithm by “acting it out.”
